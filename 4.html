LESSON 18

أنواع Input — Input Types

text · email · password · number · date · tel · url · search — المدخل المناسب في المكان المناسب

📖
ما هو عنصر <input>؟

عنصر <input> هو أكثر عناصر النماذج استخداماً في HTML. يُنشئ حقلاً تفاعلياً يتيح للمستخدم إدخال البيانات. وما يميّزه هو خاصية type التي تُغيّر شكله وسلوكه كلياً.

عنصر <input> وسم ذاتي الإغلاق (self-closing) — لا يحتاج وسم إغلاق.

⚙️
أنواع type الأساسية
type="text" نص عادي — أي محتوى
type="email" بريد إلكتروني — يتحقق من الصيغة
type="password" كلمة مرور — يُخفي الأحرف
type="number" أرقام فقط مع أسهم للزيادة
type="date" منتقي تاريخ (calendar picker)
type="tel" رقم هاتف — لوحة أرقام في الجوال
type="url" رابط — يتحقق من صيغة https://
type="search" بحث — يُضيف زر مسح (×)
type="time" منتقي وقت — ساعة ودقيقة
type="month" منتقي شهر وسنة
type="week" منتقي أسبوع
type="submit" زر إرسال النموذج
🎨
معاينة حية — أنواع Input الأساسية
💻
الخصائص المشتركة لجميع input
الخاصيةالوظيفةمثال
placeholderنص تلميحي يظهر قبل الكتابةplaceholder="اكتب هنا..."
requiredحقل إلزامي — لا يمكن إرسال النموذج بدونهrequired
disabledيُعطّل الحقل — لا يمكن التفاعل معهdisabled
readonlyللقراءة فقط — يُظهر القيمة دون تعديلreadonly
valueالقيمة الافتراضية التي تظهر في الحقلvalue="محمد"
nameاسم الحقل الذي يُرسَل مع البياناتname="username"
idمعرّف فريد لربطه بـ labelid="email"
maxlengthأقصى عدد من الأحرفmaxlength="50"
min / maxأقل وأعلى قيمة (للأرقام والتواريخ)min="1" max="100"
autocompleteتفعيل أو تعطيل الإكمال التلقائيautocomplete="off"
HTML — Input Types 
<!-- نص عادي مع placeholder -->
<input type="text" name="name" placeholder="أدخل اسمك" required>

<!-- بريد إلكتروني -->
<input type="email" name="email" placeholder="example@gmail.com">

<!-- كلمة مرور -->
<input type="password" name="pass" minlength="8" required>

<!-- رقم بين 1 و 100 -->
<input type="number" name="age" min="1" max="100">

<!-- اختيار تاريخ -->
<input type="date" name="birthday">

<!-- حقل بقيمة افتراضية -- للقراءة فقط -->
<input type="text" value="المغرب" readonly>
✏️
تمرين
1
نموذج تعريف شخصي
  1. أنشئ نموذجاً يحتوي على: الاسم الكامل، البريد الإلكتروني، تاريخ الميلاد، رقم الهاتف، كلمة المرور
  2. اجعل جميع الحقول إلزامية (required)
  3. أضف placeholder مناسب لكل حقل
  4. حدد maxlength="50" لحقل الاسم
الحل 
<form>
  <input type="text"
         name="fullname"
         placeholder="الاسم الكامل"
         maxlength="50"
         required>

  <input type="email"
         name="email"
         placeholder="example@gmail.com"
         required>

  <input type="date"
         name="birthdate"
         required>

  <input type="tel"
         name="phone"
         placeholder="+212 6XX XXX XXX"
         required>

  <input type="password"
         name="password"
         placeholder="كلمة المرور (8 أحرف على الأقل)"
         minlength="8"
         required>

  <button type="submit">إرسال</button>
</form>

// ملخص الدرس

  • خاصية type تُحدد نوع وسلوك الحقل — اختر النوع الصحيح دائماً
  • type="email" و type="url" تتحقق من صيغة البيانات تلقائياً
  • type="password" يُخفي النص أثناء الكتابة
  • required يجعل الحقل إلزامياً قبل إرسال النموذج
  • placeholder نص مؤقت يختفي عند الكتابة — لا يُرسَل مع البيانات
  • name خاصية أساسية — بدونها لا تُرسَل البيانات
LESSON 19

نموذج تسجيل الدخول — Login Form

form · action · method · label · input — كيف تبني نموذج دخول احترافياً من الصفر

📖
عنصر <form> — الحاوية الأم

كل النماذج تبدأ بعنصر <form> الذي يُحدد أين وكيف تُرسَل البيانات. الخاصيتان الأهم هما:

action: عنوان الصفحة التي تستقبل البيانات (URL الخادم).

method: طريقة الإرسال — إما GET (تظهر البيانات في URL) أو POST (مخفية في جسم الطلب — للبيانات الحساسة).

📖
عنصر <label> — التسمية

<label> يُعرّف تسمية لحقل معين. عند ربطه بالحقل عبر for و id، يصبح النقر على التسمية يُنشّط الحقل مباشرة — وهو ممتاز للوصولية (Accessibility).

HTML — ربط label بـ input 
<!-- الطريقة الصحيحة: for="id" -->
<label for="email">البريد الإلكتروني</label>
<input type="email" id="email" name="email">

<!-- أو تغليف الحقل داخل label -->
<label>
  البريد الإلكتروني
  <input type="email" name="email">
</label>

ليس لديك حساب؟ سجّل الآن

الكود الكامل لنموذج الدخول
HTML — Login Form 
<form action="/login" method="POST">

  <!-- البريد الإلكتروني -->
  <label for="email">البريد الإلكتروني</label>
  <input
    type="email"
    id="email"
    name="email"
    placeholder="example@gmail.com"
    required
    autocomplete="email"
  >

  <!-- كلمة المرور -->
  <label for="password">كلمة المرور</label>
  <input
    type="password"
    id="password"
    name="password"
    minlength="8"
    required
    autocomplete="current-password"
  >

  <!-- زر الإرسال -->
  <button type="submit">دخول</button>

</form>
🔒
GET vs POST: استخدم دائماً method="POST" مع بيانات حساسة ككلمات المرور. GET يضع البيانات في URL ويمكن رؤيتها في سجل المتصفح.
✏️
تمرين
1
بنِ نموذج دخول كامل
  1. أنشئ <form> مع action="/login" و method="POST"
  2. أضف label لكل حقل مرتبط بـ for و id
  3. أضف حقل email و password كلاهما required
  4. أضف زر submit بنص "تسجيل الدخول"
  5. أضف رابط <a> أسفل النموذج "نسيت كلمة المرور؟"
الحل 
<form action="/login" method="POST">

  <label for="email">البريد الإلكتروني</label>
  <input type="email" id="email"
         name="email" required>

  <label for="pass">كلمة المرور</label>
  <input type="password" id="pass"
         name="password" required>

  <button type="submit">تسجيل الدخول</button>

</form>

<a href="/forgot-password">نسيت كلمة المرور؟</a>

// ملخص الدرس

  • <form> هي الحاوية الأم لجميع عناصر النموذج
  • action تُحدد عنوان الخادم الذي يستقبل البيانات
  • method="POST" للبيانات الحساسة — method="GET" للبحث والفلترة
  • <label for="id"> يُحسّن تجربة المستخدم والوصولية
  • name خاصية إلزامية — بدونها لا تُرسَل قيمة الحقل
LESSON 20

القائمة المنسدلة — Select و Option

select · option · optgroup · multiple · size — اختيار واحد أو متعدد من قائمة

📖
ما هو <select>؟

عنصر <select> يُنشئ قائمة منسدلة (dropdown) يختار منها المستخدم خياراً واحداً أو أكثر. داخله تضع عناصر <option> تمثّل الخيارات المتاحة.

الخاصية value في كل option هي القيمة التي تُرسَل إلى الخادم، بينما النص بين الوسمين هو ما يراه المستخدم.

🎨
معاينة — أنواع Select
اختيار بسيط
اختيار متعدد (multiple)
Ctrl + النقر لاختيار أكثر من خيار
💻
الكود — Select مع Optgroup
HTML — Select & Option 
<!-- قائمة بسيطة -->
<select name="country" id="country">
  <option value="">-- اختر بلداً --</option>
  <option value="ma">المغرب</option>
  <option value="eg" selected>مصر</option>
  <option value="sa" disabled>السعودية (غير متاح)</option>
</select>

<!-- مع مجموعات optgroup -->
<select name="city">
  <optgroup label="المغرب">
    <option value="cas">الدار البيضاء</option>
    <option value="rab">الرباط</option>
  </optgroup>
  <optgroup label="مصر">
    <option value="cai">القاهرة</option>
    <option value="alx">الإسكندرية</option>
  </optgroup>
</select>

<!-- اختيار متعدد -->
<select name="skills[]" multiple size="4">
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <option value="js">JavaScript</option>
</select>
الخاصيةتُوضع علىالوظيفة
selectedoptionيجعله الخيار الافتراضي المحدد
disabledoption / selectيُعطّل الخيار أو كامل القائمة
multipleselectيسمح بتحديد أكثر من خيار
sizeselectعدد الخيارات المرئية بدون تمرير
valueoptionالقيمة المُرسَلة — مختلفة عن النص المعروض
✏️
تمرين
1
نموذج بيانات شخصية مع Select
  1. أضف قائمة "الجنس" مع خيارين: ذكر / أنثى
  2. أضف قائمة "المستوى التعليمي" مع 4 خيارات مقسمة بـ optgroup (مدرسة / جامعة)
  3. أضف الخيار الأول فارغاً "-- اختر --" بدون value
الحل 
<!-- قائمة الجنس -->
<label for="gender">الجنس</label>
<select id="gender" name="gender">
  <option value="">-- اختر --</option>
  <option value="m">ذكر</option>
  <option value="f">أنثى</option>
</select>

<!-- المستوى التعليمي مع optgroup -->
<label for="edu">المستوى التعليمي</label>
<select id="edu" name="education">
  <option value="">-- اختر --</option>
  <optgroup label="المدرسة">
    <option value="primary">ابتدائي</option>
    <option value="secondary">ثانوي</option>
  </optgroup>
  <optgroup label="الجامعة">
    <option value="bachelor">بكالوريوس</option>
    <option value="master">ماجستير</option>
  </optgroup>
</select>

// ملخص الدرس

  • <select> ينشئ قائمة منسدلة — <option> يُمثّل كل خيار
  • value في option هو ما يُرسَل إلى الخادم — النص هو ما يراه المستخدم
  • selected يجعل الخيار محدداً افتراضياً
  • optgroup يُجمّع الخيارات تحت عنوان دون أن يكون قابلاً للاختيار
  • multiple يسمح بتحديد أكثر من خيار عبر Ctrl + النقر

LESSON 22

التعليقات في HTML

<!-- تعليق --> — كيف توثّق كودك وتُخفي عناصر مؤقتاً

📖
ما هو التعليق؟

التعليق في HTML هو نص يكتبه المطوّر داخل الكود لأغراض التوثيق والشرح. لا يظهر في المتصفح ولا يؤثر على تصميم الصفحة — لكنه يُرى بوضوح في أدوات المطوّر (DevTools).

التعليق يبدأ بـ <!-- وينتهي بـ --> ويمكن أن يمتد على عدة أسطر.

// ملخص الدرس

  • التعليق يبدأ بـ <!-- وينتهي بـ --> ولا يظهر في المتصفح

LESSON 23

Radio و Checkbox

type="radio" · type="checkbox" — الاختيار الأحادي والمتعدد في النماذج

📖
الفرق بين Radio و Checkbox
🔘 Radio — اختيار واحد
  • يُحدد خياراً واحداً فقط من المجموعة
  • يُجمَّع بـ name متطابق
  • اختيار أحدهم يُلغي الآخر
  • مثال: الجنس، النوع، حالة الاشتراك
☑️ Checkbox — اختيار متعدد
  • يُحدد صفراً أو أكثر من الخيارات
  • كل صندوق مستقل — بـ name خاص به
  • لا يؤثر تحديد أحدها على الباقين
  • مثال: الاهتمامات، الموافقة على الشروط

الخاصيةتعمل معالوظيفة
checkedradio, checkboxيجعله محدداً افتراضياً عند تحميل الصفحة

valueradio, checkboxالقيمة المُرسَلة عند تحديده

✏️
تمرين
1
استبيان بسيط
  1. أنشئ مجموعة radio لاختيار مستوى الخبرة: مبتدئ / متوسط / محترف
  2. اجعل "مبتدئ" محدداً افتراضياً (checked)
  3. أضف مجموعة checkboxes: الأدوات التي تستخدمها (VS Code, Chrome, Figma, Git)
  4. أضف checkbox للموافقة على الشروط required

</label>

// ملخص الدرس

  • radio لاختيار واحد من المجموعة — يُجمَّع بنفس name
  • checkbox لاختيار صفر أو أكثر — كل واحد له name مستقل
  • checked يحدد الخيار الافتراضي عند تحميل الصفحة
  • value هو ما يُرسَل إلى الخادم — غير checked لا يُرسَل شيء
  • required على checkbox يُجبر المستخدم على تحديده (مفيد لقبول الشروط)
LESSON 24

رفع الملفات — Input File

type="file" · accept · multiple — بناء حقل رفع الملفات والصور

📖
حقل رفع الملف

type="file" يُنشئ حقل رفع ملف يفتح متصفح الملفات عند النقر عليه. يمكن تقييده بأنواع محددة من الملفات عبر خاصية accept، وتفعيل رفع أكثر من ملف بخاصية multiple.

ولإرسال الملفات يجب استخدام enctype="multipart/form-data" في عنصر <form>.

🎨
معاينة — مناطق رفع الملفات
📁
اسحب الملف هنا أو انقر للاختيار
accept="*" — أي نوع ملف
🖼️
رفع صور متعددة
accept="image/*" · multiple
💻
الكود — أنواع accept
HTML — Input File 
<!-- ملف واحد — أي نوع -->
<input type="file" name="document">

<!-- صور فقط -->
<input type="file" name="photo"
       accept="image/*">

<!-- صور JPEG و PNG فقط -->
<input type="file" name="avatar"
       accept="image/jpeg, image/png">

<!-- ملفات PDF فقط -->
<input type="file" name="cv"
       accept="application/pdf">

<!-- فيديو فقط -->
<input type="file" name="video"
       accept="video/*">

<!-- ملفات متعددة -->
<input type="file" name="photos[]"
       accept="image/*" multiple>

<!-- النموذج يجب أن يحوي enctype -->
<form
  action="/upload"
  method="POST"
  enctype="multipart/form-data"
>
  <input type="file" name="file" required>
  <button type="submit">رفع</button>
</form>
قيمة acceptما يقبله
image/*جميع تنسيقات الصور (jpg, png, gif, webp...)
video/*جميع تنسيقات الفيديو
audio/*جميع تنسيقات الصوت
application/pdfملفات PDF فقط
.docx,.pdfامتدادات محددة
image/jpeg,image/pngJPEG و PNG فقط
🔐
enctype إلزامي لرفع الملفات: بدون enctype="multipart/form-data" على عنصر <form>، لن تُرسَل الملفات إلى الخادم — فقط اسمها.
✏️
تمرين
1
نموذج رفع ملف السيرة الذاتية
  1. أنشئ <form> مع method="POST" و enctype="multipart/form-data"
  2. أضف حقل نص للاسم (required)
  3. أضف حقل file يقبل PDF فقط (required)
  4. أضف حقل file آخر لصورة الملف الشخصي — يقبل صور فقط
  5. زر submit بنص "إرسال الطلب"
الحل 
<form
  action="/apply"
  method="POST"
  enctype="multipart/form-data"
>
  <label for="name">الاسم الكامل</label>
  <input type="text" id="name"
         name="name" required>

  <label for="cv">السيرة الذاتية (PDF)</label>
  <input type="file" id="cv"
         name="cv"
         accept="application/pdf"
         required>

  <label for="photo">الصورة الشخصية</label>
  <input type="file" id="photo"
         name="photo"
         accept="image/*">

  <button type="submit">إرسال الطلب</button>
</form>

// ملخص الدرس

  • type="file" يُنشئ حقل رفع ملفات يفتح متصفح الملفات
  • accept يُقيّد أنواع الملفات المسموح بها — الحماية الكاملة تكون في الخادم
  • multiple يسمح برفع أكثر من ملف في آنٍ واحد
  • enctype="multipart/form-data" إلزامي على <form> لإرسال الملفات فعلياً
  • name="files[]" مع القوسين عند رفع ملفات متعددة لاستقبالها كمصفوفة
LESSON 25

Input متقدم — color · range · hidden

color · range · hidden · datetime-local · number step — الحقول المتخصصة والمتقدمة

📖
حقول Input المتخصصة

إلى جانب الأنواع الأساسية التي تعلمناها في الدرس 18، يوفر HTML أنواعاً متقدمة وممتعة لحالات استخدام محددة: منتقي الألوان، شريط التمرير، الحقول المخفية، وغيرها.

🎨
معاينة حية — Input المتقدم
type="color" منتقي الألوان
type="range" شريط التمرير
60%
type="number" + step
يزيد بمقدار 5 في كل ضغطة
type="datetime-local"
type="week" اختيار أسبوع
type="month" اختيار شهر
💻
الكود الكامل
HTML — Advanced Input Types 
<!-- منتقي الألوان -->
<label for="bg-color">لون الخلفية</label>
<input type="color" id="bg-color"
       name="bg_color" value="#F4631E">

<!-- شريط التمرير -->
<label for="vol">مستوى الصوت</label>
<input type="range" id="vol"
       name="volume"
       min="0" max="100" value="50">

<!-- رقم بخطوة محددة -->
<label for="price">السعر (بالخمسينات)</label>
<input type="number" id="price"
       name="price"
       min="0" max="1000" step="50">

<!-- تاريخ ووقت معاً -->
<label for="appt">موعد الاجتماع</label>
<input type="datetime-local" id="appt"
       name="appointment">

<!-- حقل مخفي — لا يراه المستخدم -->
<input type="hidden"
       name="user_id"
       value="42">

<!-- datalist — اقتراحات مع حرية الكتابة -->
<label for="browser">متصفحك المفضل</label>
<input type="text" id="browser"
       name="browser"
       list="browsers">
<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
</datalist>
⚙️
type="hidden" — الحقل المخفي

type="hidden" حقل لا يظهر للمستخدم لكنه يُرسَل مع باقي بيانات النموذج. يُستخدم لتمرير بيانات ثابتة يحتاجها الخادم مثل: معرف المستخدم، رمز CSRF للحماية، أو اسم الصفحة المصدر.

💡
datalist: يُضيف قائمة اقتراحات لحقل نص — المستخدم يمكنه اختيار من القائمة أو كتابة نصه الخاص. يختلف عن select الذي يُجبر على اختيار قيمة محددة.
✏️
تمرين
1
نموذج تخصيص الملف الشخصي
  1. أضف منتقي لون لتخصيص لون ثيم الملف الشخصي
  2. أضف range لتحديد مستوى الخصوصية (0 إلى 100)
  3. أضف حقل datetime-local لتحديد موعد التفعيل
  4. أضف hidden field يحمل اسم الصفحة = "profile-settings"
  5. أضف datalist لحقل "المدينة" مع 5 مدن مغربية
الحل 
<!-- لون الثيم -->
<label for="theme">لون الثيم</label>
<input type="color" id="theme"
       name="theme_color" value="#3B82F6">

<!-- مستوى الخصوصية -->
<label for="priv">مستوى الخصوصية</label>
<input type="range" id="priv"
       name="privacy"
       min="0" max="100" value="50">

<!-- موعد التفعيل -->
<label for="activate">موعد التفعيل</label>
<input type="datetime-local" id="activate"
       name="activation_date">

<!-- حقل مخفي -->
<input type="hidden"
       name="page"
       value="profile-settings">

<!-- مدينة مع datalist -->
<label for="city">المدينة</label>
<input type="text" id="city"
       name="city" list="cities">
<datalist id="cities">
  <option value="الدار البيضاء">
  <option value="الرباط">
  <option value="فاس">
  <option value="مراكش">
  <option value="الراشيدية">
</datalist>

// ملخص الدرس

  • type="color" ينشئ منتقي ألوان يعيد قيمة hex مثل #FF5500
  • type="range" ينشئ شريط تمرير — استخدم min و max و step
  • type="hidden" يُرسَل مع النموذج لكن لا يراه المستخدم
  • type="datetime-local" يجمع التاريخ والوقت في منتقٍ واحد
  • datalist يُضيف اقتراحات لحقل نص مع بقاء حرية الكتابة
  • step يُحدد وحدة الزيادة في الأرقام وشريط التمرير
