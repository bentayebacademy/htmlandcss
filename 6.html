BDI — مشكلة الكتابة المختلطة

Bi-Directional Isolation — عزل النص ذي الاتجاه المختلف داخل نص عربي

📖
المشكلة

عند كتابة كلمات إنجليزية أو أرقام داخل جملة عربية، يحاول المتصفح تخمين اتجاه كل كلمة — وأحياناً يخطئ ويعكس الترتيب. عنصر <bdi> يعزل النص ويمنع هذا الخطأ.

🎨
مقارنة — مع وبدون <bdi>
LIVE PREVIEW

بدون bdi: المستخدم أحمد Al-Rashidi حصل على 95 نقطة

مع bdi: المستخدم أحمد Al-Rashidi حصل على 95 نقطة

💻
الكود
HTML — BDI
<!-- مشكلة: اسم أجنبي قد ينعكس -->
<p>المتصبق: <bdi>Ahmed Al-Rashidi</bdi> — 95 نقطة</p>

<!-- قائمة مستخدمين بأسماء متعددة اللغات -->
<ul>
  <li>المرتبة 1: <bdi>محمد</bdi></li>
  <li>المرتبة 2: <bdi>John Smith</bdi></li>
  <li>المرتبة 3: <bdi>李明</bdi></li>
</ul>
💡
هناك أيضاً <bdo dir="ltr"> الذي يُجبر اتجاهاً محدداً. أما <bdi> فيكتشف الاتجاه تلقائياً ويعزله.

// ملخص

  • <bdi> يعزل النص ويمنع تشويه الاتجاه في النصوص المختلطة
  • ضروري عند عرض أسماء مستخدمين أو محتوى ديناميكي متعدد اللغات
  • <bdo dir="ltr/rtl"> يُجبر اتجاهاً صريحاً
LESSON 37

HTML Semantic — الوسوم الدلالية

header · nav · main · article · section · aside · footer — بنية صفحة احترافية

📖
ما هي الوسوم الدلالية؟

الوسوم الدلالية (Semantic HTML) هي عناصر تصف معنى المحتوى لا مجرد شكله. <header> يخبر المتصفح ومحركات البحث أن هذا ترويسة — وليس مجرد div. نتيجة: SEO أفضل، وصولية أفضل، كود أكثر قابلية للفهم.

🎨
هيكل صفحة دلالية كاملة
<header>
الشعار · عنوان الموقع · شعار
<nav>
روابط التنقل الرئيسية
<main>
<article>
مقالة · منشور مدونة
<section>
قسم موضوعي ضمن المقالة
<aside>
شريط جانبي · إعلانات · وصلات ذات صلة
<footer>
حقوق الملكية · روابط قانونية · تواصل
💻
الكود الكامل
HTML — Semantic Structure
<header>
  <h1>اسم الموقع</h1>
  <nav>
    <a href="/">الرئيسية</a>
    <a href="/about">من نحن</a>
  </nav>
</header>

<main>
  <article>
    <h2>عنوان المقالة</h2>
    <section>
      <h3>القسم الأول</h3>
      <p>محتوى...</p>
    </section>
  </article>
  <aside>
    <h3>مقالات ذات صلة</h3>
  </aside>
</main>

<footer>
  <p>© 2025 جميع الحقوق محفوظة</p>
</footer>
الوسميُستخدم لـ
<header>ترويسة الصفحة أو القسم — الشعار والعنوان
<nav>روابط التنقل الرئيسية
<main>المحتوى الرئيسي — مرة واحدة فقط في الصفحة
<article>محتوى مستقل بذاته (مقالة، منشور، تعليق)
<section>قسم موضوعي ضمن المحتوى
<aside>محتوى جانبي — شريط جانبي أو ملاحظة
<footer>تذييل الصفحة أو القسم
<figure> / <figcaption>صورة مع وصف
<time>تاريخ أو وقت
<mark>نص مُظلَّل / مميَّز

// ملخص

  • الوسوم الدلالية تُحسّن SEO لأن محركات البحث تفهم هيكل الصفحة
  • تُحسّن الوصولية لقارئات الشاشة للمكفوفين
  • تجعل الكود أسهل قراءةً وصيانةً من div soup
  • استخدم <main> مرة واحدة فقط في كل صفحة
LESSON 38

ربط HTML مع CSS — الطرق الثلاث

External · Internal · Inline — أيها تختار ومتى؟

📖
الطرق الثلاث لإضافة CSS
📄
External — ملف خارجي
ملف .css منفصل مربوط بـ <link> — الأفضل دائماً للمشاريع الحقيقية
📝
Internal — داخل الصفحة
داخل <style> في <head> — مقبول لصفحة واحدة أو نماذج أولية
🖊️
Inline — داخل العنصر
خاصية style= مباشرة — تجنبه في الإنتاج — صعب الصيانة
💻
الكود — الطرق الثلاث
HTML — CSS الثلاث طرق
<head>
  <!-- 1. External: الأفضل ✅ -->
  <link rel="stylesheet" href="style.css">

  <!-- 2. Internal: مقبول أحياناً ⚠️ -->
  <style>
    body { background: #F7F5F1; }
    h1   { color: #F4631E; }
  </style>
</head>
<body>
  <!-- 3. Inline: تجنبه ❌ -->
  <h1 style="color: orange; font-size: 2rem;">
    عنوان
  </h1>
</body>
🎯
الأولوية (Specificity): Inline > Internal > External — الـ Inline يتغلب على الجميع. لذلك تجنبه لأنه يصعّب تعديل التصميم لاحقاً.

// ملخص

  • External CSS: الأفضل — ملف منفصل يُشارَك بين صفحات متعددة
  • Internal CSS: مقبول للنماذج والصفحات المستقلة
  • Inline CSS: تجنبه في المشاريع الحقيقية — يُعقّد الصيانة
  • الأولوية: Inline تفوز على الجميع

Fieldset و Legend

تجميع حقول النموذج في مجموعات مصنّفة بصرياً

📖
ما هما؟

<fieldset> يُجمّع حقولاً ذات صلة في إطار. <legend> يُضيف عنواناً لهذا الإطار يظهر في حدوده العلوية.

🎨
معاينة حية
LIVE PREVIEW
📋 البيانات الشخصية
📧 بيانات التواصل
💻
الكود
HTML — Fieldset & Legend
<form>
  <fieldset>
    <legend>📋 البيانات الشخصية</legend>
    <label for="name">الاسم</label>
    <input type="text" id="name" name="name">
  </fieldset>

  <fieldset>
    <legend>📧 بيانات التواصل</legend>
    <label for="email">البريد</label>
    <input type="email" id="email" name="email">
  </fieldset>

  <button type="submit">إرسال</button>
</form>

// ملخص

  • <fieldset> يُجمّع حقول ذات صلة في إطار بصري
  • <legend> يُضيف عنواناً يظهر في الحد العلوي للإطار
  • يُحسّن الوصولية — قارئات الشاشة تُعلن اسم المجموعة قبل كل حقل
LESSON 41

id و Class — تحديد العناصر

الفرق بين المعرّف الفريد والصنف القابل للتكرار

📖
id vs class
🔑 id — معرّف فريد
  • فريد لكل صفحة — لا يتكرر
  • في CSS: #id-name
  • للـ JavaScript والـ anchor links
  • أولوية أعلى من class في CSS
🏷️ class — صنف متكرر
  • يمكن تطبيقه على عناصر متعددة
  • في CSS: .class-name
  • عنصر واحد يمكنه حمل أكثر من class
  • الأكثر استخداماً في المشاريع
💻
الكود
HTML + CSS — id & class
<!-- HTML -->
<div id="main-header" class="card featured">
  ترويسة رئيسية
</div>
<div class="card">بطاقة عادية</div>
<div class="card">بطاقة عادية</div>

/* CSS */
/* id — مرة واحدة فقط */
#main-header { background: #111; color: white; }

/* class — يطبّق على الجميع */
.card     { border: 1px solid #ddd; padding: 16px; }
.featured { border-color: #F4631E; }

/* anchor link بـ id */
Anchor Links بـ id
<!-- رابط للقفز لقسم معين -->
<a href="#contact">تواصل معنا</a>

<!-- القسم المستهدف -->
<section id="contact">
  <h2>تواصل معنا</h2>
</section>

// ملخص

  • id فريد لكل صفحة — class قابل للتكرار على عناصر متعددة
  • عنصر يمكنه حمل عدة classes مفصولة بمسافة
  • id في CSS يبدأ بـ # وclass بـ .
  • id مفيد للـ anchor links وJavaScript — class للتصميم
LESSON 42

Favicon — أيقونة الموقع

الأيقونة الصغيرة في تبويب المتصفح ونتائج البحث

📖
ما هو Favicon؟

Favicon هي الأيقونة الصغيرة التي تظهر في تبويب المتصفح، نتائج بحث Google، وقائمة المفضلة. تُضاف داخل <head> بوسم <link>.

💻
الكود الكامل
HTML — Favicon
<head>
  <!-- .ico — الأساسي، يعمل في كل المتصفحات -->
  <link rel="icon" href="favicon.ico">

  <!-- PNG — أفضل جودة -->
  <link rel="icon"
        type="image/png"
        href="favicon-32x32.png"
        sizes="32x32">

  <!-- لأجهزة Apple -->
  <link rel="apple-touch-icon"
        href="apple-touch-icon.png"
        sizes="180x180">

  <!-- SVG — قابل للتكبير بلا جودة -->
  <link rel="icon"
        type="image/svg+xml"
        href="favicon.svg">
</head>
💡
لإنشاء favicon من أي صورة مجاناً: استخدم favicon.io أو realfavicongenerator.net — يولّدان جميع الأحجام تلقائياً.

// ملخص

  • Favicon تظهر في التبويب والمفضلة ونتائج البحث
  • ico. الأكثر توافقاً — PNG وSVG للأجهزة الحديثة
  • apple-touch-icon لأجهزة iOS عند إضافة الموقع للشاشة الرئيسية
LESSON 43

Meta SEO — تحسين محركات البحث

description · keywords · robots · author — كيف تظهر صفحتك في Google

📖
Meta SEO الأساسية

هذه الوسوم تُخبر محركات البحث بمحتوى الصفحة وكيفية فهرستها. description يظهر مباشرة تحت العنوان في نتائج Google.

💻
الكود الكامل
HTML — Meta SEO
<head>
  <!-- العنوان — 50-60 حرف مثالي -->
  <title>دورة HTML وCSS بالعربية | ISPITS</title>

  <!-- الوصف — 150-160 حرف، يظهر في Google -->
  <meta name="description"
        content="تعلّم HTML وCSS من الصفر حتى
        الاحتراف — 60 درساً مجاناً بالعربية">

  <!-- الكلمات المفتاحية (أقل أهمية اليوم) -->
  <meta name="keywords"
        content="HTML, CSS, تعلم البرمجة, عربي">

  <!-- المؤلف -->
  <meta name="author" content="زكرياء">

  <!-- توجيهات لمحركات البحث -->
  <meta name="robots"
        content="index, follow">

  <!-- الرابط الأساسي للصفحة -->
  <link rel="canonical"
        href="https://mysite.com/html-course">
</head>
الوسمالوظيفةالأهمية
<title>عنوان الصفحة في Google والتبويب🔥 حاسم
descriptionالوصف الذي يظهر تحت العنوان في Google🔥 مهم جداً
robotsindex/noindex — هل تُفهرسها محركات البحث؟⚠️ مهم
canonicalيمنع تكرار المحتوى في محركات البحث⚠️ مهم
keywordsكلمات مفتاحية — تجاهلها Google اليوم💤 ضعيف

// ملخص

  • <title> أهم عنصر SEO — يظهر في Google وفي التبويب
  • description يظهر تحت العنوان في نتائج البحث — اجعله مقنعاً
  • robots="index, follow" يسمح لـ Google بالفهرسة والمتابعة
  • canonical يمنع تكرار المحتوى بين صفحات متعددة

Code · Abbr · Dfn

وسوم النصوص التقنية والاختصارات والتعريفات

🎨
معاينة حية
LIVE PREVIEW

استخدم وسم <p> للفقرات.

منظمة W3C هي من تضع معايير الويب.

HTML هي لغة ترميز النص التشعبي المستخدمة لبناء صفحات الويب.

اضغط Ctrl+S للحفظ.

💻
الكود
HTML — Code, Abbr, Dfn, Kbd
<!-- code: كود برمجي مضمّن -->
<p>استخدم <code>console.log()</code> لطباعة النتائج.</p>

<!-- abbr: اختصار مع tooltip -->
<p>
  <abbr title="HyperText Markup Language">
    HTML
  </abbr>
   لغة ترميز صفحات الويب.
</p>

<!-- dfn: تعريف مصطلح -->
<p>
  <dfn>CSS</dfn>
   هي لغة تصميم صفحات الويب.
</p>

<!-- kbd: مفاتيح لوحة المفاتيح -->
<p>للحفظ: <kbd>Ctrl</kbd> + <kbd>S</kbd></p>

<!-- samp: ناتج برنامج -->
<p>الناتج: <samp>Hello, World!</samp></p>

// ملخص

  • <code> لكود برمجي مضمّن داخل جملة — مع <pre> للكتل
  • <abbr title="..."> اختصار مع tooltip عند التمرير
  • <dfn> أول تعريف لمصطلح في النص
  • <kbd> لمفاتيح لوحة المفاتيح — <samp> لناتج البرنامج
LESSON 45

Open Graph — مشاركة فيسبوك

og:title · og:description · og:image — كيف تظهر صفحتك عند مشاركتها

📖
ما هو Open Graph؟

عند مشاركة رابط موقعك على Facebook أو LinkedIn أو WhatsApp، تظهر بطاقة مع صورة وعنوان ووصف. هذه البطاقة تُبنى من وسوم Open Graph التي تضعها في <head>.

🎨
معاينة بطاقة المشاركة
🌐
mysite.com
دورة HTML وCSS بالعربية | ISPITS
تعلّم HTML وCSS من الصفر حتى الاحتراف — 60 درساً مجاناً بالعربية مع المحاضر زكرياء
💻
الكود الكامل
HTML — Open Graph Tags
<head>
  <!-- Open Graph الأساسية -->
  <meta property="og:type"
        content="website">
  <meta property="og:url"
        content="https://mysite.com/html-course">
  <meta property="og:title"
        content="دورة HTML وCSS بالعربية">
  <meta property="og:description"
        content="60 درساً من الصفر للاحتراف">
  <meta property="og:image"
        content="https://mysite.com/og-image.jpg">

  <!-- أبعاد الصورة الموصى بها: 1200×630 -->
  <meta property="og:image:width"  content="1200">
  <meta property="og:image:height" content="630">
  <meta property="og:locale"       content="ar_MA">
</head>

// ملخص

  • Open Graph يتحكم بشكل الرابط عند مشاركته على التواصل الاجتماعي
  • og:image الأبعاد المثلى: 1200×630 بيكسل
  • بدون og:image قد يختار Facebook صورة عشوائية من الصفحة
  • استخدم og:validators.opengraph.me لاختبار البطاقة
LESSON 46

رابط التحميل — download

خاصية download في <a> — تحميل ملف بدلاً من فتحه

💻
الكود
HTML — Download Link
<!-- تحميل مع اسم ملف مخصص -->
<a href="files/report.pdf"
   download="التقرير-السنوي-2025.pdf">
  ⬇️ تحميل التقرير
</a>

<!-- تحميل بالاسم الأصلي للملف -->
<a href="cv.pdf" download>
  📄 تحميل السيرة الذاتية
</a>

<!-- فتح في تبويب جديد (بدون تحميل) -->
<a href="cv.pdf" target="_blank">
  👁️ معاينة السيرة الذاتية
</a>

<!-- تحميل صورة -->
<a href="photo.jpg"
   download="صورتي.jpg">
  🖼️ تحميل الصورة
</a>
⚠️
خاصية download تعمل فقط مع ملفات من نفس النطاق (same-origin). لن تعمل مع روابط لمواقع خارجية.

// ملخص

  • download بدون قيمة يحمّل الملف باسمه الأصلي
  • download="اسم.pdf" يُغيّر اسم الملف المحمّل
  • تعمل فقط مع same-origin — لا مع روابط خارجية
  • target="_blank" لفتح بتبويب جديد بدون تحميل
LESSON 47

Blockquote و Center

اقتباسات نصية وتوسيط المحتوى

🎨
معاينة
LIVE PREVIEW
"البرمجة ليست عن ما تعرفه — بل عن ما يمكنك معرفته"
— مجهول

نص في المنتصف باستخدام CSS text-align: center

💻
الكود
HTML — Blockquote & Center
<!-- اقتباس مع مصدر -->
<blockquote cite="https://source.com">
  <p>البرمجة فن حل المشكلات.</p>
  <footer>
    — <cite>كتاب البرمجة الاحترافية</cite>
  </footer>
</blockquote>

<!-- اقتباس داخل سطر (inline) -->
<p>
  كما قال ستيف جوبز 
  <q>Stay hungry, stay foolish</q>
</p>

<!-- توسيط بـ CSS (لا تستخدم <center> القديمة) -->
<div style="text-align: center;">
  <img src="logo.png" alt="الشعار">
  <p>نص في المنتصف</p>
</div>
⚠️
وسم <center> قديم ومهجور في HTML5 — استخدم text-align: center في CSS.

// ملخص

  • <blockquote> للاقتباسات الطويلة — <q> للاقتباسات القصيرة inline
  • cite= يُشير لمصدر الاقتباس — <cite> يُسمّي المصدر بصرياً
  • <center> مهجور — استخدم CSS text-align: center

Contenteditable و Script

جعل أي عنصر قابلاً للتعديل — وإضافة JavaScript مضمّن

رسائل الخطأ في النماذج

HTML5 Form Validation — التحقق المدمج بدون JavaScript

📖
التحقق المدمج في HTML5

HTML5 يوفّر تحققاً أصيلاً من البيانات بدون JavaScript — فقط بإضافة خصائص على حقول النموذج. المتصفح يعرض رسائل خطأ تلقائية عند محاولة إرسال النموذج بيانات غير صحيحة.

🎨
جرّب — اضغط إرسال بدون تعبئة
LIVE FORM VALIDATION
💻
خصائص التحقق
HTML — Form Validation
<!-- حقل إلزامي -->
<input type="text" required>

<!-- طول النص -->
<input type="text" minlength="3" maxlength="50">

<!-- نطاق الأرقام -->
<input type="number" min="18" max="99">

<!-- نمط regex -->
<input
  type="text"
  pattern="[0-9]{6}"
  title="أدخل 6 أرقام فقط"
>

<!-- رسالة خطأ مخصصة عبر JS -->
<input
  type="email"
  oninvalid="this.setCustomValidity(
    'الرجاء إدخال بريد إلكتروني صحيح')"
  oninput="this.setCustomValidity('')"
>

<!-- تعطيل التحقق لكامل النموذج -->
<form novalidate>...</form>

// ملخص

  • required يمنع إرسال النموذج إذا كان الحقل فارغاً
  • pattern يقبل regex للتحقق من صيغة محددة
  • title يُظهر رسالة خطأ مخصصة عند عدم مطابقة pattern
  • novalidate على <form> يُعطّل التحقق التلقائي كاملاً
  • setCustomValidity() من JS لرسائل خطأ عربية مخصصة

  • مهم لمواقع React وVue التي تعتمد على JS كلياً

Meta UTF-8 — تعمق

ما هو الترميز ولماذا UTF-8 الحل الأمثل؟

📖
ما هو الترميز (Encoding)؟

الحاسوب يخزّن كل شيء كأرقام (0 و 1). الترميز هو الجدول الذي يحدد: أي رقم يقابل أي حرف. مثلاً في ASCII، الرقم 65 = A. لكن ASCII لا يتضمن الحروف العربية ولا الصينية ولا الرموز التعبيرية.

UTF-8 (Unicode Transformation Format) يُمثّل أكثر من 140,000 حرف من جميع لغات العالم والرموز — هو المعيار العالمي اليوم.

⚙️
مقارنة الترميزات
الترميزاللغات المدعومةالحجم
ASCIIالإنجليزية فقط — 128 حرف1 بايت
ISO-8859-6العربية + الإنجليزية1 بايت
UTF-16جميع اللغات2-4 بايت
UTF-8جميع اللغات + الرموز 🌍1-4 بايت متغير
