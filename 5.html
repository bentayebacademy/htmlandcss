LESSON 26

Metaview · Lang · Dir · UTF-8

الإعدادات الأساسية لكل صفحة HTML — الترميز واللغة والاتجاه والعرض

📖
لماذا هذه الإعدادات مهمة؟

قبل أي شيء آخر في صفحتك، هناك أربعة إعدادات يجب أن تكون في كل ملف HTML تكتبه. هذه الإعدادات تخبر المتصفح بكيفية قراءة الصفحة وعرضها بشكل صحيح — بدونها قد تظهر الحروف العربية مكسورة أو تتصرف الصفحة بشكل غريب على الهواتف.

💻
الكود الكامل — رأس كل صفحة HTML
HTML — إعدادات الصفحة الأساسية 
<!DOCTYPE html>

<!-- 1. اللغة واتجاه الكتابة -->
<html lang="ar" dir="rtl">

<head>
  <!-- 2. ترميز الأحرف UTF-8 —  أول شيء في head -->
  <meta charset="UTF-8">

  <!-- 3. Viewport — يجعل الصفحة متجاوبة مع الموبايل -->
  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  >

  <title>عنوان الصفحة</title>
</head>
⚙️
شرح كل إعداد
الإعدادالوظيفةالقيم الشائعة
langلغة الصفحة — للمتصفح ومحركات البحث وقارئات الشاشةar · en · fr
dirاتجاه الكتابة في كامل الصفحةrtl عربي · ltr إنجليزي
charset="UTF-8"ترميز الأحرف — يدعم العربية والإنجليزية والرموز كلهادائماً UTF-8
viewportيجعل الصفحة تتكيف مع عرض شاشة الجهازwidth=device-width, initial-scale=1.0
🔥
قاعدة ذهبية: ضع <meta charset="UTF-8"> كأول سطر داخل <head> — قبل أي شيء آخر — لأن المتصفح يبدأ قراءة الترميز من أول بايت.
🔍
Viewport بالتفصيل
❌ بدون viewport
  • الصفحة تبدو مصغّرة جداً على الموبايل
  • المستخدم يحتاج للـ zoom يدوياً
  • فشل في Google Mobile Friendly
✅ مع viewport
  • العرض يساوي عرض شاشة الجهاز
  • initial-scale=1.0 لا تكبير ابتدائي
  • ضروري لأي تصميم Responsive

// ملخص الدرس

  • lang="ar" dir="rtl" على <html> لصفحات اللغة العربية
  • charset="UTF-8" أول شيء في <head> — يمنع تكسّر الحروف
  • viewport ضروري لكل صفحة — بدونه الموبايل لا يعرض الصفحة صحيح
  • هذه الثلاثة موجودة في كل صفحة HTML احترافية بدون استثناء
LESSON 27

ربط الملفات ببعضها

كيف تربط صفحات HTML مع CSS وJavaScript والصور والملفات الخارجية

📖
ربط CSS خارجي

ملف CSS منفصل يُربط بوسم <link> داخل <head>. هذا هو الأسلوب الاحترافي الذي يتيح مشاركة ملف CSS واحد بين صفحات متعددة.

HTML — ربط CSS 
<head>
  <!-- ربط ملف CSS محلي -->
  <link rel="stylesheet" href="style.css">

  <!-- ربط CSS من مجلد فرعي -->
  <link rel="stylesheet" href="css/main.css">

  <!-- ربط خط من Google Fonts -->
  <link
    rel="stylesheet"
    href="https://fonts.googleapis.com/css2?family=Tajawal"
  >
</head>
💻
ربط JavaScript
HTML — ربط JavaScript 
<!-- ربط JS في نهاية body — الأفضل -->
<body>
  <!-- محتوى الصفحة -->

  <script src="script.js"></script>
  <script src="js/app.js"></script>
</body>

<!-- أو في head مع defer (لا يوقف تحميل الصفحة) -->
<head>
  <script src="app.js" defer></script>
</head>
⚙️
جميع طرق الربط
الوسميُستخدم لـيُوضع في
<link rel="stylesheet">ملفات CSS الخارجية<head>
<script src="...">ملفات JavaScriptنهاية <body> أو <head defer>
<img src="...">الصورداخل <body>
<link rel="icon">أيقونة الموقع Favicon<head>
<a href="page.html">الربط بصفحات أخرىداخل <body>
💡
defer vs async: defer ينتظر اكتمال تحميل HTML ثم يُشغّل السكريبت بالترتيب. async يُحمّل ويُشغّل بمجرد الجاهزية بدون ترتيب — استخدم defer للأمان.
✏️
تمرين
1
هيكل مشروع كامل
  1. أنشئ هيكل HTML يربط: ملف CSS اسمه style.css من مجلد css/
  2. أضف ربط لخط Tajawal من Google Fonts
  3. أضف Favicon يشير إلى images/favicon.ico
  4. أضف ملف JS اسمه main.js في نهاية body
الحل 
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>موقعي</title>

  <!-- Favicon -->
  <link rel="icon" href="images/favicon.ico">

  <!-- Google Font -->
  <link rel="stylesheet"
        href="https://fonts.googleapis.com/css2?family=Tajawal">

  <!-- CSS الخاص بالمشروع -->
  <link rel="stylesheet" href="css/style.css">
</head>
<body>

  <!-- محتوى الصفحة -->

  <script src="main.js"></script>
</body>
</html>

// ملخص الدرس

  • <link rel="stylesheet"> يربط CSS — يُوضع في <head>
  • <script src> يربط JS — الأفضل في نهاية <body> أو مع defer
  • الترتيب في <head>: charset أولاً، ثم viewport، ثم title، ثم الروابط
  • defer أفضل من async لأنه يحافظ على ترتيب تشغيل الملفات
LESSON 28

عنصر Meter

قياس قيمة ضمن نطاق محدد — شريط مؤشر HTML5 المدمج

📖
ما هو <meter>؟

عنصر <meter> يُعرض قيمة ضمن نطاق معروف — مثل نسبة اكتمال مهمة، مستوى بطارية، أو درجة في اختبار. يختلف عن <progress> في أن meter يُستخدم لقيمة ثابتة محددة وليس عملية جارية.

🎨
معاينة حية
مستوى عالٍ 90%
مستوى متوسط 60%
مستوى منخفض 20%
درجة / 100 78
💻
الكود والخصائص
HTML — Meter Element 
<!-- meter بسيط -->
<meter value="0.7">70%</meter>

<!-- meter مع min وmax -->
<meter
  value="75"
  min="0"
  max="100"
>75 من 100</meter>

<!-- meter مع تحديد المناطق -->
<meter
  value="78"
  min="0"
  max="100"
  low="40"
  high="80"
  optimum="95"
  title="درجة الاختبار"
>
  78 من 100
</meter>
الخاصيةالوظيفة
valueالقيمة الحالية — إلزامية
min / maxالحد الأدنى والأعلى للنطاق (افتراضي: 0 و 1)
lowالحد الذي تُعدّ دونه القيمة منخفضة — يُلوّن المتصفح بالأصفر
highالحد الذي تُعدّ فوقه القيمة عالية
optimumالقيمة المثلى — يُحدد المتصفح اللون بناءً على موقعها

// ملخص الدرس

  • <meter> لقيم ثابتة ضمن نطاق — <progress> لعمليات جارية
  • low و high و optimum تُخبر المتصفح كيف يُلوّن الشريط تلقائياً
  • النص داخل <meter> يظهر فقط في المتصفحات التي لا تدعمه
  • استخدمه لعرض: الدرجات، نسبة الاكتمال، مستوى البطارية، الإشغال
LESSON 29

عنصر Textarea

حقل نص متعدد الأسطر — التعليقات والرسائل والأوصاف الطويلة

📖
Textarea مقابل Input

<textarea> حقل نص قابل للتمديد يقبل نصوصاً طويلة متعددة الأسطر. يختلف عن <input type="text"> في أنه:

📝 <input type="text">
  • سطر واحد فقط
  • وسم ذاتي الإغلاق
  • للاسم والبريد وكلمة المرور
📋 <textarea>
  • أسطر متعددة — قابل للتمديد
  • له وسم فتح وإغلاق
  • للتعليقات والرسائل والأوصاف
🎨
معاينة حية
LIVE PREVIEW — Textarea
💻
الكود والخصائص
HTML — Textarea 
<!-- textarea أساسي -->
<label for="msg">رسالتك</label>
<textarea
  id="msg"
  name="message"
  rows="5"
  cols="40"
  placeholder="اكتب رسالتك هنا..."
  required
></textarea>

<!-- بقيمة افتراضية — النص يُكتب بين الوسمين -->
<textarea name="bio" rows="4">أهتم بتطوير الويب والتصميم...</textarea>

<!-- بدون إمكانية التمديد -->
<textarea
  name="note"
  rows="3"
  style="resize: none;"
  maxlength="200"
></textarea>
الخاصيةالوظيفة
rowsعدد الأسطر المرئية — يُحدد الارتفاع الأولي
colsعدد الأعمدة — الأفضل التحكم بالعرض عبر CSS
placeholderنص تلميحي يختفي عند الكتابة
maxlengthالحد الأقصى لعدد الأحرف
resize: none (CSS)يمنع المستخدم من تمديد الحقل
⚠️
تنبيه مهم: القيمة الافتراضية لـ <textarea> تُكتب بين الوسمين — لا توجد خاصية value كما في <input>. وأي مسافة أو سطر جديد بين الوسمين سيظهر كنص.

// ملخص الدرس

  • <textarea> للنصوص الطويلة — له وسم فتح وإغلاق
  • rows يُحدد الارتفاع المرئي — تحكّم في العرض بـ CSS
  • القيمة الافتراضية تُكتب بين الوسمين وليس في خاصية value
  • resize: none في CSS يمنع المستخدم من تغيير حجم الحقل
LESSON 30

عنصر iFrame

تضمين صفحات وموارد خارجية داخل صفحتك — النوافذ المضمّنة

📖
ما هو <iframe>؟

<iframe> (Inline Frame) يُنشئ نافذة مضمّنة داخل الصفحة تعرض صفحة ويب أخرى أو مصدراً خارجياً كامل الاستقلالية. يُستخدم لتضمين خرائط Google وفيديوهات YouTube ولوحات المدفوعات وغيرها.

💻
الكود والخصائص
HTML — iFrame 
<!-- iframe أساسي -->
<iframe
  src="https://example.com"
  width="600"
  height="400"
  title="صفحة مضمّنة"
></iframe>

<!-- iframe YouTube -->
<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="فيديو تعليمي"
  allowfullscreen
></iframe>

<!-- iframe بدون border وبخاصية sandbox -->
<iframe
  src="page.html"
  width="100%"
  height="500"
  frameborder="0"
  sandbox="allow-scripts allow-same-origin"
></iframe>
الخاصيةالوظيفة
srcعنوان الصفحة أو المورد المراد تضمينه
width / heightأبعاد الإطار — أرقام أو نسب مئوية
titleوصف الإطار للوصولية — إلزامي للـ Accessibility
allowfullscreenيسمح بعرض المحتوى كاملاً على الشاشة
frameborder="0"يُزيل الحد حول الإطار
sandboxيُقيّد صلاحيات المحتوى المضمّن للأمان
loading="lazy"تحميل كسول — لا يُحمَّل حتى يظهر في الشاشة
🎨
تمثيل بصري
// الصفحة الرئيسية
<iframe>
📄 صفحة أخرى مستقلة
لها CSS وJS خاصين بها
⚠️
X-Frame-Options: كثير من المواقع كـ Google وFacebook تمنع تضمينها في iframe بسبب هذا الهيدر الأمني. يمكنك فقط تضمين مواقع تسمح بذلك صراحةً — كـ YouTube بكود embed الخاص بها.

// ملخص الدرس

  • <iframe> يُنشئ نافذة مستقلة تعرض صفحة أو مورداً خارجياً
  • title إلزامي لأسباب الوصولية — لا تتركه فارغاً
  • loading="lazy" يُحسّن أداء الصفحة — يؤخّر تحميل الـ iframe
  • sandbox يُضيف طبقة أمان على المحتوى المضمّن
  • استخدامه مع YouTube وGoogle Maps والبطاقات التفاعلية شائع جداً
LESSON 31

عنصر hr — الخط الفاصل

Horizontal Rule — الفاصل البصري بين الأقسام

📖
ما هو <hr>؟

<hr> (Horizontal Rule) عنصر ذاتي الإغلاق يُنشئ خطاً أفقياً فاصلاً. في HTML5 أصبح له معنى دلالي — يُشير إلى تحوّل موضوعي أو انتقال بين قسمين مختلفين من المحتوى. شكله الافتراضي بسيط، لكن CSS يُتيح تخصيصه بالكامل.

🎨
أنواع hr بالـ CSS

نص قبل الفاصل — القسم الأول من المحتوى

↑ hr افتراضي

فاصل ملوّن بلون البرتقالي

↑ border-top: 2px solid orange

فاصل منقط

↑ border-top: 2px dashed

فاصل سميك داكن

↑ border-top: 6px solid + border-radius

💻
الكود
HTML + CSS — hr 
<!-- hr بسيط -->
<hr>

<!-- تخصيص hr بـ CSS -->
<style>
  hr {
    border: none;
    border-top: 2px solid #F4631E;
    margin: 32px 0;
  }

  /* hr منقط */
  hr.dashed {
    border-top: 2px dashed #6B6560;
  }

  /* hr بتدرج */
  hr.gradient {
    border: none;
    height: 1px;
    background: linear-gradient(
      to right, transparent,
      #F4631E, transparent
    );
  }
</style>
💡
لإنشاء hr بتدرج لونه، استخدم border: none مع background: linear-gradient(...) و height: 1px — لأن border-top لا يقبل التدرج مباشرة.

// ملخص الدرس

  • <hr> فاصل أفقي ذاتي الإغلاق — وسم دلالي في HTML5
  • شكله الافتراضي يختلف بين المتصفحات — استخدم CSS للتحكم الكامل
  • border: none ثم border-top لتجاوز تنسيق المتصفح الافتراضي
  • يُستخدم للفصل بين الأقسام، نهاية المقالات، وفواصل بصرية
LESSON 32

عنصر pre — النص المُهيَّأ مسبقاً

Preformatted Text — عرض الكود والمسافات والأسطر كما هي

📖
ما هو <pre>؟

المتصفح عادةً يُدمج كل المسافات المتعددة والأسطر الجديدة في صفحة HTML إلى مسافة واحدة. عنصر <pre> يُلغي هذا السلوك — يعرض النص بالضبط كما هو مكتوب، بخط ثابت العرض (monospace) مع الحفاظ على كل المسافات والأسطر.

🎨
مقارنة — مع وبدون <pre>
❌ بدون <pre>
النص هنا مهما كتبت مسافات بين الكلمات ستظهر مسافة واحدة فقط
✅ مع <pre>
النص هنا     المسافات
  تُحفظ
    كما هي بالضبط!
💻
الاستخدامات الشائعة
HTML — pre Element 
<!-- عرض كود برمجي -->
<pre>
function greet(name) {
  console.log("Hello, " + name);
  return true;
}
</pre>

<!-- مع <code> للدلالة -->
<pre>
  <code>
  <div class="container">
    <p>محتوى</p>
  </div>
  </code>
</pre>

<!-- ASCII Art أو قصيدة -->
<pre>
  /\_____/\
 /  o   o  \
( ==  ^  == )
 )         (
(           )
</pre>
💡
<pre> + <code>: الجمع بينهما هو الممارسة الصحيحة — <pre> للحفاظ على التنسيق، و<code> لإعطاء دلالة دلالية بأن المحتوى كود برمجي. تجدهما دائماً معاً في توثيق البرمجة.

// ملخص الدرس

  • <pre> يعرض النص كما هو — يحافظ على المسافات والأسطر
  • يستخدم خطاً ثابت العرض (monospace) تلقائياً
  • يُستخدم عادةً مع <code> لعرض أكواد برمجية
  • أي مسافة أو سطر جديد بين وسمي <pre> سيظهر للمستخدم
LESSON 33

div و span

الحاويتان العامتان — اللبنة الأساسية لتنظيم وتصميم صفحات الويب

📖
الفرق الجوهري

كلاهما حاوية بلا معنى دلالي — وجودهما الوحيد هو التجميع والتنظيم لأغراض CSS وJavaScript.

📦 <div> — Block Element
  • يأخذ سطراً كاملاً لوحده
  • يمكن تحديد عرضه وارتفاعه
  • لتجميع أقسام وعناصر كاملة
  • أساس بناء الـ Layout
🔤 <span> — Inline Element
  • يقع داخل السطر مع النص
  • لا يُحدث سطراً جديداً
  • لتنسيق جزء من نص
  • لا عرض أو ارتفاع بدون CSS
🎨
توضيح بصري
<div> — يأخذ سطراً كاملاً

هذا نص عادي، وهنا <span> داخل السطر يستمر النص بعده في نفس السطر. يمكن تنسيق كلمة محددة بشكل مختلف.

<div> آخر — بلوك جديد

كل <div> يبدأ في سطر جديد تلقائياً.

💻
الكود الكامل
HTML — div & span 
<!-- div لتجميع قسم -->
<div class="card">
  <div class="card-header">
    <h2>عنوان البطاقة</h2>
  </div>
  <div class="card-body">
    <p>محتوى البطاقة هنا...</p>
  </div>
</div>

<!-- span لتنسيق جزء من نص -->
<p>
  السعر الأصلي 
  <span style="text-decoration:line-through;color:gray">
    200 درهم
  </span>
   الآن فقط 
  <span style="color:#F4631E;font-weight:bold">
    150 درهم!
  </span>
</p>
🎯
قاعدة مهمة: استخدم div و span فقط عندما لا يوجد وسم HTML دلالي مناسب. للتنقل استخدم <nav>، للمقالة <article>، للترويسة <header> — هذه أفضل دلالياً وللـ SEO.
✏️

  3. اجعل السعر يحتوي على span للسعر الأصلي مشطوباً وآخر للسعر الجديد

LESSON 34

Google Maps داخل HTML

تضمين خريطة تفاعلية في أي صفحة ويب مجاناً

📖
كيف تضمّن Google Maps؟

Google Maps توفّر كود iframe يمكن نسخه مباشرة وتضمينه في أي صفحة HTML — بدون مفتاح API ومجاناً. تجده من خلال: البحث عن الموقع في Google Maps ← النقر على "مشاركة" ← تبويب "تضمين خريطة".

💻
كود التضمين
HTML — Google Maps Embed 
<!-- الكود المباشر من Google Maps -->
<iframe
  src="https://www.google.com/maps/embed?pb=!1m18!1m12!
1m3!1d3323.8...!2d-5.0!3d34.0!2m3!1f0!2f0!3f0!3m2!
1i1024!2i768!4f13.1!3m3!1m2!..."
  width="600"
  height="400"
  style="border:0"
  allowfullscreen=""
  loading="lazy"
  referrerpolicy="no-referrer-when-downgrade"
  title="موقع المؤسسة على الخريطة"
></iframe>

<!-- تكبير الخريطة لتأخذ عرض الصفحة كاملاً -->
<iframe
  src="https://www.google.com/maps/embed?..."
  width="100%"
  height="450"
  style="border:0; border-radius: 12px;"
  loading="lazy"
  title="خريطة الموقع"
></iframe>
💡
خطوات الحصول على كود التضمين
  • 1افتح maps.google.com وابحث عن الموقع المراد
  • 2انقر على اسم الموقع لفتح لوحته الجانبية
  • 3انقر على أيقونة مشاركة (Share)
  • 4اختر تبويب "تضمين خريطة" (Embed a map)
  • 5انسخ كود iframe وألصقه في صفحتك

// ملخص الدرس

  • Google Maps تُوفّر كود iframe جاهزاً بدون API Key
  • استخدم width="100%" لجعل الخريطة متجاوبة
  • loading="lazy" ضروري لتجنب إبطاء تحميل الصفحة
  • style="border:0" يُزيل الحد الافتراضي حول الإطار
  • أضف title وصفياً لأسباب الوصولية
LESSON 35

مسارات الملفات — File Paths

Absolute · Relative · Root-relative — كيف تحدد موقع أي ملف بدقة

📖
أنواع المسارات الثلاثة

المسار (Path) هو العنوان الذي يُحدد مكان ملف ما. في HTML، تحتاجه في كل مرة تربط صورة أو ملف CSS أو صفحة أخرى. هناك ثلاثة أنواع يجب أن تتقنها.

🎨
هيكل مشروع للمثال
هيكل المشروع 
my-project/
├── index.html         ← نحن هنا
├── about.html
├── css/
│   └── style.css
├── images/
│   ├── logo.png
│   └── hero.jpg
└── pages/
    └── contact.html
💻
النوع 1 — المسار النسبي (Relative Path)

ينطلق من موقع الملف الحالي — الأكثر استخداماً في المشاريع.

نفس المجلد href="about.html" ملف في نفس المجلد
مجلد فرعي src="images/logo.png" نزول لمجلد فرعي
مجلد فرعي href="css/style.css" CSS من مجلد css/
مجلد أعلى src="../images/logo.png" من pages/ للرجوع لأعلى
مجلدين أعلى src="../../logo.png" صعود مستويين
💻
النوع 2 — المسار المطلق (Absolute Path)

العنوان الكامل بما فيه البروتوكول — يُستخدم للموارد الخارجية.

مسار مطلق 
<!-- مسار مطلق خارجي -->
<img src="https://example.com/images/logo.png">
<link href="https://fonts.googleapis.com/css2?...">

<!-- لا تستخدم المسار المطلق المحلي في الموقع -->
<!-- ❌ سيء: C:/Users/zakarya/project/images/logo.png -->
<!-- ✅ استخدم النسبي أو root-relative -->
💻
النوع 3 — Root-Relative (من جذر الموقع)

يبدأ بـ / وينطلق من جذر الموقع — مفيد في المشاريع الكبيرة مع خادم.

Root-Relative Paths 
<!-- يبدأ دائماً من جذر الموقع -->
<link href="/css/style.css" rel="stylesheet">
<img  src="/images/logo.png" alt="الشعار">
<a    href="/pages/contact.html">تواصل معنا</a>
النوعيبدأ منمتى تستخدمه
about.htmlالملف الحالينفس المجلد — الأشيع
../images/x.pngالملف الحالي صعوداًالعودة لمجلد أعلى
/images/x.pngجذر الموقعمشاريع على خادم
https://...الإنترنتموارد خارجية
⚠️
../ يعني "اصعد مجلداً واحداً". إذا كنت في pages/contact.html وتريد الصورة من images/ فالمسار: ../images/logo.png
✏️
تمرين
1
أكمل المسارات الصحيحة
project/
├── index.html      ← أنت هنا
├── css/style.css
├── js/app.js
└── assets/
    └── img/logo.png
  1. اربط ملف CSS في <head>
  2. اربط ملف JS في نهاية <body>
  3. أضف صورة الشعار
الحل
<head>
  <!-- CSS من مجلد css/ -->
  <link rel="stylesheet" href="css/style.css">
</head>
<body>

  <!-- الصورة من assets/img/ -->
  <img src="assets/img/logo.png" alt="الشعار">

  <!-- JS من مجلد js/ -->
  <script src="js/app.js"></script>
</body>

// ملخص الدرس

  • المسار النسبي ينطلق من موقع الملف الحالي — الأشيع استخداماً
  • ../ يعني "اصعد مجلداً" — يمكن تكرارها ../../ للصعود أكثر
  • المسار الكامل https:// للموارد الخارجية فقط
  • /path يبدأ من جذر الموقع — يعمل فقط على خادم وليس ملفات محلية
  • خطأ المسار سبب رقم 1 لعدم ظهور الصور والـ CSS
