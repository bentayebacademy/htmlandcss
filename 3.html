LESSON 12

عرض ملفات PDF

تضمين ملفات PDF مباشرة داخل صفحة الويب بطرق مختلفة

📖
الطرق المتاحة لعرض PDF

هناك ثلاث طرق رئيسية لعرض ملفات PDF في HTML:

🖼️
<iframe>
الأكثر توافقاً وتحكماً — يعرض الـ PDF داخل إطار مضمّن
📎
<embed>
وسم مباشر لتضمين الملفات — أبسط لكن أقل تحكماً
🔗
رابط تحميل
الأبسط — رابط <a> يفتح أو يحمّل الملف مباشرة
💻
الطريقة الأولى — <iframe>

هذه هي الطريقة الموصى بها لعرض PDF — تعطيك تحكماً كاملاً في الأبعاد وتعمل في جميع المتصفحات.

HTML — PDF داخل iframe 
<!-- عرض PDF بالعرض الكامل -->
<iframe
  src="document.pdf"
  width="100%"
  height="600px"
  title="وثيقة PDF"
>
  <!-- نص بديل إذا لم يدعم المتصفح iframe -->
  متصفحك لا يدعم عرض PDF.
  <a href="document.pdf">اضغط هنا لتحميله</a>
</iframe>
// معاينة بصرية لما يبدو عليه iframe
<iframe>

HTML — PDF بـ embed

💻
الطريقة الثالثة — رابط تحميل
HTML — رابط فتح أو تحميل

<a href="document.pdf" target="_blank">
  📄 افتح الوثيقة
</a>

<!-- يُحمّل الـ PDF مباشرة (خاصية download) -->

</a>

LESSON 13

عنصر <embed>

تضمين محتوى خارجي مباشرة — ملفات، صوت، فيديو، مواقع

📖
ما هو <embed>؟

وسم <embed> هو وسم ذاتي الإغلاق يُضمّن محتوى خارجياً مباشرةً داخل الصفحة. يمكنه تضمين: ملفات PDF، ملفات فلاش قديمة، وسائط، أو حتى صفحات ويب أخرى.

الفرق الرئيسي عن iframe: <embed> ذاتي الإغلاق ولا يمكنه احتواء نص بديل، بينما <iframe> له وسم إغلاق ويمكن وضع محتوى بديل داخله.

💻
الكود وأنواع الاستخدام
HTML — embed لأنواع مختلفة 
<!-- تضمين PDF -->
<embed
  src="document.pdf"
  type="application/pdf"
  width="800"
  height="500"
>

<!-- تضمين صورة SVG -->
<embed
  src="graphic.svg"
  type="image/svg+xml"
  width="400"
  height="300"
>

<!-- تضمين صفحة ويب (مثل iframe) -->
<embed
  src="https://example.com"
  type="text/html"
  width="100%"
  height="400"
>
⚙️
خصائص <embed>
الخاصيةالوظيفةمثال
srcمسار أو رابط الملف — مطلوبsrc="file.pdf"
typeنوع MIME للملفtype="application/pdf"
widthعرض منطقة العرضwidth="800"
heightارتفاع منطقة العرضheight="500"
⚙️
أنواع MIME الشائعة:
PDF → application/pdf
SVG → image/svg+xml
HTML → text/html
MP4 → video/mp4
⚖️
embed مقابل iframe — متى تستخدم أيهما؟
✅ استخدم iframe
  • تضمين صفحات ويب كاملة
  • تريد نصاً بديلاً للمتصفحات القديمة
  • تحتاج تحكماً أكبر (sandbox، allow)
  • تضمين YouTube وخرائط Google
🟡 استخدم embed
  • تضمين ملفات PDF بشكل مباشر
  • تضمين SVG أو وسائط مباشرة
  • لا تحتاج لنص بديل
  • تريد كوداً أقصر وأبسط

// ملخص الدرس

  • <embed> وسم ذاتي الإغلاق لتضمين ملفات خارجية
  • يحتاج src و type (نوع MIME) و width و height
  • لا يدعم نصاً بديلاً — استخدم iframe إذا احتجت ذلك
  • الأفضل لـ PDF وSVG، أما المواقع فـ iframe أفضل
LESSON 14

تضمين YouTube — iFrame

كيف تضمّن فيديوهات يوتيوب وغيرها في صفحتك بالشكل الصحيح

📖
لماذا iframe لـ YouTube؟

يوتيوب لا يسمح لك بتحميل الفيديو مباشرةً — بدلاً من ذلك يعطيك كود تضمين (Embed Code) يعمل عبر <iframe> يُحمّل الفيديو من خوادم يوتيوب مباشرةً.

هذا الأسلوب يعمل مع يوتيوب، فيميو، ساوند كلاود، وأي منصة وسائط أخرى.

💻
كيف تحصل على رابط التضمين
  • ١
    افتح فيديو يوتيوب ← اضغط على زر Share (مشاركة)
  • ٢
    اضغط على Embed (تضمين)
  • ٣
    انسخ الكود — ستجد https://www.youtube.com/embed/VIDEO_ID
  • ٤
    الصق الكود في صفحتك وعدّل الأبعاد كما تريد
💻
الكود الكامل
HTML — تضمين يوتيوب 
<!-- الشكل الأساسي -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560"
  height="315"
  title="عنوان الفيديو"
  allowfullscreen
></iframe>

<!-- نسخة متجاوبة (Responsive) -->
<div style="position:relative; padding-bottom:56.25%; height:0; overflow:hidden;">
  <iframe
    src="https://www.youtube.com/embed/VIDEO_ID"
    style="position:absolute; top:0; left:0; width:100%; height:100%;"
    title="عنوان الفيديو"
    allowfullscreen
  ></iframe>
</div>
📐
56.25% في النسخة المتجاوبة = 9/16 × 100 — وهو نسبة حجم الفيديو القياسية (16:9). هذه الحيلة تجعل الفيديو يتمدد ويتقلص مع حجم الشاشة بشكل صحيح تماماً.
⚙️
معاملات YouTube الإضافية

يمكنك التحكم في سلوك فيديو يوتيوب بإضافة معاملات لرابط الـ src:

HTML — معاملات YouTube 
<!-- يبدأ من ثانية 30، مع تشغيل تلقائي، بدون شريط التحكم -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID?start=30&autoplay=1&controls=0"
  width="560"
  height="315"
  allowfullscreen
></iframe>
المعاملالوظيفةمثال
autoplay=1تشغيل تلقائي (يشترط mute=1)?autoplay=1&mute=1
controls=0إخفاء شريط التحكم?controls=0
loop=1تكرار الفيديو?loop=1&playlist=ID
start=Nالبدء من ثانية N?start=60
mute=1كتم الصوت?mute=1
rel=0عدم عرض فيديوهات مقترحة في النهاية?rel=0
✏️
تمرين
1
صفحة دروس فيديو
  1. اختر أي فيديو من يوتيوب وانسخ معرّفه (VIDEO_ID)
  2. أضف الفيديو إلى صفحتك بعرض 100% وارتفاع 400px
  3. أضف allowfullscreen وعنوان title مناسب
  4. جرّب إضافة ?rel=0 لمنع الفيديوهات المقترحة
الحل 
<h2>درس HTML الأول</h2>

<iframe
  src="https://www.youtube.com/embed/dQw4w9WgXcQ?rel=0"

  height="400"
  title="درس HTML للمبتدئين"
  allowfullscreen

// ملخص الدرس

  • يوتيوب يُضمَّن عبر iframe باستخدام رابط /embed/VIDEO_ID
  • allowfullscreen يتيح للمستخدم توسيع الفيديو لملء الشاشة
  • النسخة المتجاوبة تستخدم padding-bottom: 56.25%
  • يمكن التحكم في السلوك بمعاملات URL مثل ?rel=0&controls=0
  • أضف title دائماً لأسباب إمكانية الوصول والـ SEO
LESSON 15

عنصر <base>

تحديد العنوان الأساسي لجميع الروابط في الصفحة دفعة واحدة

📖
ما هو <base>؟

وسم <base> يُوضع داخل <head> ويحدد العنوان الأساسي (Base URL) الذي تُبنى عليه جميع الروابط النسبية في الصفحة.

بدلاً من كتابة المسار الكامل في كل رابط وصورة، تكتبه مرة واحدة في <base> والباقي يُكمل تلقائياً.

⚠️
يُسمح بوسم <base> مرة واحدة فقط في كل صفحة، ويجب أن يكون أول شيء داخل <head> قبل أي روابط أو موارد.
💻
كيف يعمل <base>
HTML — base في head 
<head>
  <base href="https://www.mysite.com/pages/" target="_blank">
  <meta charset="UTF-8">
</head>
<body>
  <!-- بدون base: <a href="https://www.mysite.com/pages/about.html"> -->
  <!-- مع base: يصبح الرابط النسبي يكفي -->
  <a href="about.html">من نحن</a>
  <!-- يُترجم تلقائياً إلى: https://www.mysite.com/pages/about.html -->

  <img src="logo.png" alt="الشعار">
  <!-- يُترجم إلى: https://www.mysite.com/pages/logo.png -->
</body>
<base href="..."> https://mysite.com/pages/
about.html → يصبح → https://mysite.com/pages/about.html
logo.png → يصبح → https://mysite.com/pages/logo.png
#section → يبقى → #section (لا يتأثر)
⚙️
خاصية target في <base>

يمكن لـ <base> أيضاً تحديد target افتراضي لجميع روابط الصفحة. مثلاً: target="_blank" يجعل كل الروابط تفتح في تبويب جديد تلقائياً.

HTML — base مع target 
<!-- كل الروابط ستفتح في تبويب جديد -->
<base target="_blank">

<!-- أو الجمع بين href و target -->
<base
  href="https://cdn.mysite.com/assets/"
  target="_self"
>
💡
متى تستخدم <base>؟ مفيد جداً عند: استخدام CDN لجميع الأصول (صور، ملفات CSS)، أو عند بناء موقع تكون فيه جميع صفحاتك في مجلد معين.

// ملخص الدرس

  • <base> يحدد العنوان الأساسي لكل الروابط النسبية في الصفحة
  • يوضع داخل <head> — مرة واحدة فقط لكل صفحة
  • target في <base> يحدد هدف جميع الروابط افتراضياً
  • الروابط المطلقة (https://...) لا تتأثر بـ <base>
  • مفيد للمشاريع التي تستخدم CDN أو ملفات في مجلد مشترك
LESSON 16

Details و Summary

إنشاء محتوى قابل للطي والتوسيع بدون JavaScript

📖
ما هما details و summary؟

<details> يُنشئ عنصراً قابلاً للطي — يخفي محتواه افتراضياً ويظهره عند الضغط. <summary> هو عنوان ذلك العنصر — الجزء الظاهر دائماً الذي يُضغط عليه.

الجميل في هذا العنصر أنه يعمل بدون JavaScript أو CSS — HTML صافٍ 100%.

🎨
معاينة حية — اضغط لتجربة
ما هو HTML؟

HTML هو اختصار لـ HyperText Markup Language — لغة هيكل صفحات الويب.

ما الفرق بين HTML و CSS؟

HTML يبني الهيكل (عناوين، فقرات، صور)، بينما CSS يُنسّق ويُلوّن ويُجمّل ذلك الهيكل.

هل details يعمل بدون JavaScript؟ ✓

نعم! هذا العنصر مدمج في HTML5 ويعمل بشكل كامل بدون أي كود JavaScript أو CSS.

💻
الكود
HTML — details و summary 
<!-- مغلق افتراضياً -->
<details>
  <summary>اضغط هنا لرؤية المزيد</summary>
  <p>هذا المحتوى كان مخفياً وظهر بعد الضغط.</p>
</details>

<!-- مفتوح افتراضياً بخاصية open -->
<details open>
  <summary>هذا مفتوح افتراضياً</summary>
  <p>يمكن إغلاقه بالضغط على العنوان.</p>
</details>

<!-- يمكن وضع أي محتوى داخل details -->
<details>
  <summary>الإجابة النموذجية</summary>
  <ul>
    <li>النقطة الأولى</li>
    <li>النقطة الثانية</li>
  </ul>
</details>

LESSON 17

الأزرار — Buttons

type="button" · type="submit" · type="reset" — الزر المناسب في المكان المناسب

📖
وسم <button>

وسم <button> ينشئ زراً قابلاً للنقر. يختلف عن <input type="button"> في أنه يمكنه احتواء محتوى HTML (نصوص، أيقونات، صور).

الخاصية الأهم هي type التي تحدد سلوك الزر:

🎨
أنواع الأزرار — معاينة حية
⚙️
أنواع type الثلاثة
النوعالوظيفةمتى تستخدمه
type="submit" يُرسل النموذج (form) — القيمة الافتراضية زر إرسال نموذج تسجيل أو دخول
type="button" زر عادي — لا يفعل شيئاً بدون JavaScript أي زر تريد التحكم فيه بـ JS
type="reset" يُعيد تعيين جميع حقول النموذج لقيمها الأولى زر "مسح الكل" في النماذج
HTML — الأزرار الثلاثة 
<!-- زر إرسال النموذج -->
<button type="submit">إرسال</button>

<!-- زر عادي يعمل مع JavaScript -->
<button type="button" onclick="myFunction()">
  اضغط هنا
</button>

<!-- زر إعادة التعيين -->
<button type="reset">مسح الكل</button>
⚠️
مهم: القيمة الافتراضية لـ type هي submit. إذا وضعت زراً داخل نموذج بدون تحديد type سيُرسل النموذج عند الضغط عليه — حدد type="button" صراحةً إذا لم تريد ذلك.
💻
خصائص إضافية مهمة
HTML — خصائص الأزرار 
<!-- زر معطّل -->
<button type="button" disabled>غير متاح الآن</button>

<!-- زر مع أيقونة وتلميح -->
<button type="button" title="اضغط للحفظ">
  💾 حفظ
</button>

<!-- الأزرار داخل نموذج -->
<form>
  <input type="text" placeholder="اكتب هنا...">
  <button type="submit">إرسال</button>
  <button type="reset">مسح</button>
</form>
💡
button vs input type="submit":
<button> أفضل لأنه يقبل HTML داخله (أيقونات، صور، تنسيق). <input type="submit"> نص فقط ولا يقبل محتوى داخلي.
✏️
تمارين
1
نموذج بسيط بأزرار صحيحة
  1. أنشئ نموذجاً بحقل اسم وحقل بريد إلكتروني
  2. أضف زر submit لإرسال النموذج
  3. أضف زر reset لمسح الحقول
  4. أضف زر button ثالثاً مع onclick يُظهر رسالة

  >
    💾 حفظ مسودة
  </button>

// ملخص الدرس

  • <button> أفضل من input type="submit" لأنه يقبل HTML داخله
  • type="submit" يُرسل النموذج — الافتراضي إذا لم تحدد النوع
